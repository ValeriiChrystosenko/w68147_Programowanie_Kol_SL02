git hub

Kiedy pracujesz z plikami HTML w repozytorium Git, polecenie git push służy do wysyłania zatwierdzonych zmian, które zostały zapisane w lokalnym repozytorium, do zdalnego repozytorium, takiego jak GitHub, GitLab czy Bitbucket. Możesz użyć tego polecenia, aby udostępnić swoje zmiany innym członkom zespołu, zaktualizować witrynę internetową lub utworzyć kopię zapasową swojego kodu HTML.
Git push origin main
remote-odnosi się do nazwy zdalnego repozytorium, takiego jak "origin".
branch-odnosi się do nazwy gałęzi, w której dokonano zmian, na przykład "main" lub inna nazwa gałęzi.
